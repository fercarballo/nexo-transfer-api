# Definición de Listo (Definition of Ready)

Un trabajo está **listo para empezar** cuando cumple estos criterios. Evita arrancar con ambigüedad.

- [ ] **Objetivo claro:** se entiende qué problema resuelve y para quién.
- [ ] **Alcance acotado:** se sabe qué entra y qué queda fuera (backlog).
- [ ] **Criterios de aceptación:** están escritos y son verificables (idealmente como escenarios).
- [ ] **Reglas de negocio identificadas:** y su resultado esperado (incluidos los casos de error).
- [ ] **Dependencias conocidas:** datos, servicios, credenciales; si algo externo hace falta, hay
      un plan (adaptador + modo simulado).
- [ ] **Riesgos considerados:** los relevantes están en la matriz de riesgos.
- [ ] **Verificable localmente:** se puede probar sin infraestructura especial, o está documentado
      cómo levantarla.

Si un ítem no se cumple, el trabajo se refina antes de empezar, no durante.
